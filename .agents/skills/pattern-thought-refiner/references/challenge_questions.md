# Challenge Questions

Use these questions to test weak pattern drafts.

## Problem

1. What specific failure appears without this pattern?
2. Which user or system is impacted first?
3. What evidence shows this is frequent enough to deserve a pattern?

## Condition

1. What are the concrete preconditions to apply this pattern?
2. Name two scenarios where this pattern should not be used.
3. What alternative pattern would you choose in those non-conditions?

## Solution

1. Which step is most failure-prone, and how is that handled?
2. What inputs and outputs exist at each step?
3. What dependencies or tooling assumptions are required?

## Example

1. What exact workload or dataset was used?
2. What baseline did you compare against?
3. What changed after applying the pattern?

## Claims and Truthfulness

1. Which statements in this draft are proven vs assumed?
2. What would falsify each strong claim?
3. Can each claim be tied to a test, runbook, benchmark, or source?
